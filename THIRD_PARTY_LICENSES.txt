The Qt libraries are dynamically linked and provided as separate DLL files

How to replace the Qt libraries (LGPLv3 §4 compliance):

  1. Close TeamSpeak 3 completely.
  2. Navigate to the plugin's bundled-DLL directory:
       %APPDATA%\TS3Client\plugins\gs_comms\
     (this is per-user; the plugin loads its Qt copy exclusively from here)
  3. Replace any of the bundled Qt6 DLLs with your own Qt 6.x build:
       Qt6Core.dll, Qt6Gui.dll, Qt6Widgets.dll, Qt6Network.dll,
       Qt6Multimedia.dll, platforms\qwindows.dll, tls\qschannelbackend.dll
  4. Start TeamSpeak 3. The plugin will load the replaced DLLs from this
     directory at startup. The plugin uses absolute paths and does NOT
     pin DLL signatures or hashes - any binary-compatible Qt 6.x build works.

The plugin uses LoadLibraryEx with LOAD_LIBRARY_SEARCH_USER_DIRS to harden
against DLL hijacking from PATH or the current working directory; this is a
security measure and does NOT impede LGPLv3-required replaceability of the
Qt libraries themselves.
